Orange:

What is orange?

Orange is a component-based visual programming software package for data visualization, machine learning, data mining and data analysis. Orange components are called widgets and they range from simple data visualization, subset selection and pre-processing, to empirical evaluation of learning algorithms and predictive modelling.

What are we using it to accomplish?

We used Orange for the natural language processing section of the project. Orange was chosen as it is a great interactive data visualization tool that is powerful, but also easy to operate due to Orange’s use of visual programming. We used Orange to pre-process training data, create a support-vector machine model and use that model as a predictor (or classifier) to determine the accuracy of our classifier and auto label unlabelled data in order to classify the data. Orange will then output the classified data into a csv file.

Our Orange model

Components:
Import Documents: A widget that retrieves text files from folders and creates a collection of documents called a corpus. Corpus Viewer: A widget meant for viewing text files (instances of a corpus).
Preprocess Text: A widget that splits the text into smaller units (tokens), filters them, runs normalization (stemming, lemmetization), creates n-grams, and tags tokens with part-of-speech labels. Word Cloud: A widget that displays tokens in the corpus, their size denoting the frequency of the word in the corpus.
Bag of Words: A widget of which its model creates a corpus with word counts for each data instance (document). SVM: Support Vector Machine (SVM) is a machine learning technique that separates the attribute space with a hyperplane, thus maximizing the margin between the instances of different classes or class values.
Predictions: A widget that receives a dataset and one or more predictors (predictive models), and outputs the data and the prediction for the data. Data Table: A widget that receives one or more datasets in its input and displays them as a spreadsheet.
Save Data: A widget that considers a dataset provided in the input channel and saves it to a data file with a specified name.


Structure


The structure of the Orange file begins with the Import Document widget that imports the collection of required text files into a corpus (Train Data).

The Corpus Viewer is used to ensure that the correct text files were imported. The corpus is then passed into the Pre-process Text widget where the text files were tokenized, normalized and filtered. N-grams are also created in this widget.

The Word Cloud widget is used to visually identify the highest recurring token in the entire corpus. The pre-processed corpus is then sent to the Bag of Words widget where it creates a “bag” that contains the word count of each individual document.

The corpus is then passed to the SVM widget which is a machine learning widget that would maximize the margin between the instances of different values and output a predictor model. The predictor model is used in the Predictions widget.

The Predictions widget will use the predictor model on the newly inputted corpus (Test Data/Unlabelled Data) and it will output predictions on the new corpus (Test Data/Unlabelled Data). The Confusion Matrix widget is used to visually display which specific instances were misclassified.

The Data Table widget is used to display the auto labelling results as a spreadsheet. The Save Data widget is then used to export the spreadsheet to a csv file.
